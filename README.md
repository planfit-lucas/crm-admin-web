## 새 캠페인 만들기

### 1. 캠페인 정보

- **캠페인 이름**: 캠페인 식별용 이름 입력
- **채널**: Push 또는 Email 선택
  - Email 선택 시 **발송 이메일(From)** 필수 선택

### 2. 발송 대상 필터

SQL 쿼리 또는 필터 빌더로 발송 대상을 지정합니다.

```sql
SELECT u.id FROM users_user u
WHERE locale = '%KR%' OR locale = '%US%'
```

- JSON 프리뷰에서 쿼리 결과 확인 가능
- **검증** 버튼으로 쿼리 유효성 검사 후 캠페인 생성

### 3. 푸시 / 액션 설정

- **푸시 타입**: 텍스트만, 이미지 포함 등
- **액션 타입**:
  - `선택 안함` — 푸시 클릭 시 앱 홈으로 이동
  - `외부 URL` — 지정한 URL로 이동
  - `앱 네비게이션` — 앱 내 특정 스크린으로 이동 (스크린 선택 필수)
- **내부 테스트 발송 여부**: 체크 시 내부 테스트 대상에게만 발송

---

## 발송 시점 — 3가지 유형

### ⏱️ 한 번만

- 지정한 시간에 1회 발송
- 한국 시간대 기준, 타임존 자동 변환
- 용도: 공지, 이벤트 알림 등 일회성 캠페인

### 📅 정기 발송

- 반복 주기 선택: 매분 / 매시간 / 매일 / 매주 / 매월
- 발송 시간 지정
- 용도: 리텐션 푸시, 주간 리마인더

### ⚡ 이벤트 기반

- Amplitude 이벤트를 트리거로 사용
- 발송 지연: 최소 1분 ~ 최대 15분
- 용도: 행동 기반 푸시 (운동 완료, 가입 등)

## A/B 테스트 설정

1. **A/B 테스트 사용 체크** — 캠페인 생성 화면 상단의 체크박스를 활성화하면 실험 설정 패널이 열립니다
2. **실험 ID 입력** — 고유 식별자 (영문, 숫자, 언더스코어). 예: `onboarding_test`, `retention_v2`
3. **변형 & 트래픽 비율** — `+ 변형 추가`로 Variant 추가, **합계 = 100%** 필수

> ⚠️ 합계가 100%가 아니면 저장할 수 없습니다.

---

## 발송 콘텐츠 구성

- 변형별(A, B, C...) × 국가별(한국, 미국) 콘텐츠를 각각 설정
- 각 변형별로 **발송 여부**를 체크/해제하여 특정 변형만 발송하거나 제외 가능
- 내용은 Markdown 지원 (HTML 자동 생성)
- 미리보기에서 렌더링 결과 확인 가능

## 이메일 관리 & Amplitude 이벤트

### 발송 이메일 관리

- AWS SES 검증된 이메일만 사용 가능
- 등록된 발송 이메일 예시: `jhk@planfit.ai`, `hwb@planfit.ai`, `hello@planfit.ai`, `psj@planfit.ai`

### 발송 이메일 등록 절차

1. **회원가입** — 플랜핏 이메일(`@planfit.ai`)로 가입
2. **AWS 인증 메일 수신** — AWS SES에서 인증 링크가 포함된 이메일이 발송됩니다
3. **인증 링크 클릭** — 메일 내 링크를 클릭하여 AWS 인증 완료
4. **어드민에서 검증하기 클릭** — CRM Admin → 이메일 관리 → **검증하기** 버튼 클릭으로 최종 완료

### Amplitude 이벤트 관리

이벤트 기반 발송의 트리거로 사용되는 이벤트 목록:

| 이벤트 | 설명 | 상태 |
|--------|------|------|
| `on_press_workout_plan_setting_btn` | 플랜 세팅 버튼 | 활성 |
| `on_press_get_ready_workout` | 운동 준비 | 활성 |
| `on_press_save_activity_by_main` | 운동 완료 | 활성 |
| `on_test_push` | 테스트 푸시 | 활성 |

## 안내사항 — 푸시 채널

### 📱 로컬 푸시 (조회 전용)

- 앱 내에서 로컬로 발생하는 푸시
- **조회 용도로만 사용 가능**
- 발송 / 중단 등의 액션 불가

### 🖥️ 서버 푸시 (조회 · 변경)

- 서버에서 발송하는 푸시 캠페인
- **조회, 캠페인 상태 변경, 발송 콘텐츠 수정 가능**

---

## ⚠️ 주의사항 — 생성 후 변경 불가

캠페인 생성 시 확정되어 이후 수정할 수 없는 항목:

### 🔒 채널 변경 불가

푸시로 생성한 캠페인은 **이메일로 변경할 수 없습니다.** 채널은 캠페인 생성 시 확정됩니다.

### 🔒 발송 시점 변경 불가

한 번만 / 정기 발송 / 이벤트 기반 등 **발송 시점 유형은 생성 후 변경할 수 없습니다.**

> 💡 **Tip:** 변경이 필요하면 기존 캠페인을 중단하고 새로 생성하세요.

## 내부 테스트 발송이 안 올 때

푸시 또는 이메일이 수신되지 않는다면 아래 항목을 확인하세요:

1. **개발자 메뉴가 보이는지 확인** — 내부 테스트 발송 대상은 **개발자 메뉴가 활성화된 계정**만 해당됩니다. 앱에서 개발자 메뉴가 보이지 않는다면 테스트 대상이 아닙니다.
2. **이메일 도메인 확인** — 이메일 채널의 경우 **@planfit.ai** 로 끝나는 계정에만 발송됩니다. 개인 이메일(gmail 등)로는 내부 테스트 수신 불가
3. **내부 테스트 발송 체크 확인** — 캠페인 설정에서 '내부 테스트 발송' 체크박스가 활성화되어 있는지 확인
4. **푸시 알림 권한 확인** — 기기 설정에서 Planfit 앱의 푸시 알림이 허용되어 있는지 확인

### 필수 조건 요약

| 채널 | 조건 | 결과 |
|------|------|------|
| 📱 푸시 | 해당 계정에 **개발자 메뉴**가 보여야 합니다 | ✅ 수신 가능 |
| 📧 이메일 | 계정 이메일이 **@planfit.ai** 로 끝나야 합니다 | ✅ 수신 가능 |
| - | 위 조건 미충족 | ❌ 수신 불가 |
